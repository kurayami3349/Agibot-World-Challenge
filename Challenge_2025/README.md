# Challenge 2025 - 复现与学习记录

本目录主要用于记录我参与 **Agibot World Challenge 2025** 的复现工作与学习进度。

我已将比赛所需的关键资源克隆至本地，包括官方的基础模型（Baseline）、冠军模型（Champion Model）以及用于评测的仿真平台。接下来的目标是逐步跑通并复现这些模型，深入理解其背后的算法与实现。

## � 重要资源链接
- **官方数据集**: [Hugging Face - AgiBotWorldChallenge-2025](https://huggingface.co/datasets/agibot-world/AgiBotWorldChallenge-2025)
  - 包含比赛所需的仿真数据、演示轨迹等。

## �📂 目录结构

- **[eval_platform/](./eval_platform/)**
  - **描述**: 官方提供的仿真评测平台，基于 Genie Sim Benchmark。
  - **用途**: 提供高保真的仿真环境，用于评估模型在各项操作任务上的性能。

- **[baseline/](./baseline/)**
  - **描述**: 官方提供的基线模型代码。
  - **用途**: 作为复现的第一步，用于熟悉数据处理流程、模型架构以及基本的训练与推理接口。

- **[champion_x-vla/](./champion_x-vla/)**
  - **描述**: 比赛冠军团队（或高性能参考方案）的开源模型代码（X-VLA）。
  - **用途**: 深入研究高性能模型的架构设计与优化策略，尝试复现其在榜单上的表现。

## 🗓️ 复现计划

### 阶段一：环境搭建与平台熟悉
- [ ] **评测平台配置**: 完成 `eval_platform` 的环境依赖安装，确保仿真环境能正常启动。
- [ ] **官方 Demo 运行**: 跑通平台自带的示例任务，验证环境的正确性。

### 阶段二：Baseline 模型复现
- [ ] **数据准备**: 下载并整理官方数据集。
- [ ] **代码阅读**: 理解 Baseline 的数据加载、模型结构（如 UniVLA 等）及训练逻辑。
- [ ] **推理验证**: 使用官方提供的预训练权重在仿真环境中进行推理，复现基线分数。
- [ ] **训练复现**: 尝试从头训练 Baseline 模型（或进行微调），观察 Loss 收敛情况。

### 阶段三：冠军模型（Champion Model）攻关
- [ ] **架构分析**: 研读 `champion_x-vla` 代码，分析其相对于 Baseline 的改进点（如数据增强、模型结构优化、策略改进等）。
- [ ] **环境适配**: 解决冠军模型可能存在的特定环境依赖问题。
- [ ] **性能复现**: 运行冠军模型，争取在本地仿真环境中复现其优异性能。

### 阶段四：优化与总结
- [ ] **对比分析**: 总结 Baseline 与 Champion Model 的性能差异原因。
- [ ] **个人优化**: 在理解的基础上，尝试提出自己的改进思路或优化方案。
